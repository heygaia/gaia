### TL;DR
- Single required env: `NEXT_PUBLIC_API_BASE_URL`.
- Central place that validates and applies it: `frontend/src/lib/api/client.ts`.
- Add new `NEXT_PUBLIC_*` only if strictly needed; read via `process.env.NEXT_PUBLIC_*`.

### API base URL (single source)
```ts
// frontend/src/lib/api/client.ts
if (!process.env.NEXT_PUBLIC_API_BASE_URL) {
  throw new Error("Missing required environment variable: NEXT_PUBLIC_API_BASE_URL");
}

export const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_BASE_URL,
  headers: { "x-timezone": getUserTimezone() },
});

export const apiauth = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_BASE_URL,
  withCredentials: true,
  headers: { "x-timezone": getUserTimezone() },
});
```
Use `api`/`apiauth` everywhere so the base URL and headers stay consistent.

### Adding a new frontend env
- Name it `NEXT_PUBLIC_MY_SETTING`.
- Access it where needed: `process.env.NEXT_PUBLIC_MY_SETTING`.
- Validate once, close to where it’s used; avoid global scattering.
- Prefer feature toggles/config pulled from the backend when possible.

Keep it minimal: most features don’t need extra frontend envs.
